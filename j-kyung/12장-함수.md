## 12.1 함수란?
* 자바스크립트의 핵심  

### 프로그래밍 언어 속 함수의 의미
>* 일련의 과정을 `문`으로 구현하고 코드 블록으로 감싸고 하나의 실험 단위로 정의한 것
>* 함수 내부로 입력 전달받는 변수 = 매개 변수 / 입력 = 인수 / 출력 = 반환값

    * 함수 호출 시 코드 블록에 담긴 문들이 일괄적으로 실행, 반환값 반환

* 코드의 재사용 => 동일한 작업 반복 수행 시 함수 재사용하여 효율적
* 유지 보수의 편의성 상승
* 코드의 신뢰성 상승
* 코드의 가독성 향상

## 12.3 함수 리터럴
### 구성 : function 키워드, 함수 이름, 매개변수 목록, 함수 몸체
> 함수 이름
> * 식별자. 식별자 네이밍 규칙 준수
> * 함수 몸체 내에서만 참조할 수 있는 식별자
> * 생략 가능
>   * 기명 함수 = 이름이 있는 함수
>   * 무명/익명 함수 = 이름이 없는 함수  

> 매개변수 목록
> * 0개 이상의 매개변수를 소괄호로 감싸고 쉼표로 구분
> * 각 매개변수에는 함수를 호출할 때 지정한 인수가 순서대로 할당 ==> 매개변수 목록은 순서에 의미가 있음!
> * 함수 몸체 내에서 변수와 동일하게 취급. 식별자 네이밍 규칙 준수  

> 함수 몸체
> * 함수 호출되었을 때 일괄적으로 실행될 문들을 하나의 실행 단위로 정의한 코드 블록
> * 함수 호출에 의해 실행됨

* 함수 리터럴도 평가되어 값을 생성 -> 값이 객체 --> 함수는 객체!  
* but, 일반 객체는 호출할 수 없지만 함수는 호출할 수 있으며, 고유한 프로퍼티를 가짐

## 12.4 함수 정의
- 함수를 호출하기 이전에 인수를 전달받을 매개변수와 실행할 문, 반환할 값을 지정하는 것

### 함수 정의 방식 4가지
> * 함수 선언문
> * 함수 표현식
> * Function 생성자 함수
> * 화살표 함수

### 함수 선언문 & 함수 리터럴
> * 형태 동일
> * 리터럴은 함수 이름 생략 가능
> * 선언문은 함수 이름 생략 불가능
> * 선언문은 표현식이 아닌 문 (크롬 개발자 도구의 콘솔에서 실행하면 완료 값 undefined 출력)

* 단독으로 사용된 함수 리터럴은 함수 선언문으로 해석  
* 그룹 연산자() 내에 있는 함수 리터럴은 함수 리터럴 표현식으로 해석  
* 함수 선언문 사용 시 자바스크립트 엔진이 해석해서 함수 객체 생성 -> 생성된 함수 호출 위해 함수 이름과 동일한 이름의 식별자를 암묵적으로 생성, 함수 객체 할당  
* 함수 호출 - 함수 이름으로 호출 X, 함수 객체를 가리키는 `식별자`로 호출!
* 자바스크립트의 함수 = 일급 객체(값의 성질을 갖는 객체) : 함수 리터럴로 생성한 함수 객체를 변수에 할당
* 함수 표현식 - 함수 리터럴로 생성한 함수 객체를 변수에 할당

### 함수 선언문과 함수 표현식의 차이
> * 함수 선언문 - 표현식이 아닌 문  
>   * 함수 선언문으로 정의한 함수와 표현식으로 정의한 함수 생성 시점이 다름
>   * 런타임 이전에 객체 먼저 생성함
>   * 함수 선언문이 코드의 선두로! => 함수 호이스팅
> * 함수 표현식 - 표현식인 문

### 함수/변수 호이스팅 차이점
> * var 키워드로 선언된 변수 = `undefined`로 초기화
> * 함수 선언문으로 암묵적 생성된 식별자 = `함수 객체`로 초기화
>   * 함수 선언문 이전에 호출 시, 함수 호이스팅에 호출 가능
* 함수 표현식 - 변수에 할당되는 값이 함수 리터럴인 문
    * 변수 선언문 + 변수 할당문이 축약된 표현과 동일하게 동작
    * Not 함수 호이스팅! 이건 변수 호이스팅임!
* 변수 선언 - 런타임 이전에 실행 : undefined로 초기화
* 변수 할당 - 할당문이 실행되는 시점, 런타임에 평가 -> 함수 표현식도 이 때
### 생성자 함수
- 객체를 생성하는 함수
### 화살표 함수 
- 항상 익명 함수로 정의
- 생성자 함수로 사용 불가
- 기존 함수와 this 바인딩이 다름
- prototype 프로퍼티 존재 X, arguments 객체 생성 X

## 12.5
- 함수를 가리키는 식별자 + 한 쌍의 소괄호인 함수 호출 연산자
- 현재의 실행 흐름 중단, 호출된 함수로 실행 흐름 옮김
    - 매개변수에 인수가 순서대로 할당, 함수 몸체의 문들이 실행
        - 인수 : 값으로 평가될 수 있는 표현식, 함수 호출 시 지정, 개수 or 타입 제한 X, 함수 내부에서 변수와 동일하게 취급, 함수 외부에서 사용 X
- 함수 호출 -> 암묵적 매개변수 생성 -> undefined 초기화 -> 인수 순서대로 할당
> * 인수보다 매개변수가 많은 경우 인수가 할당되지 않은 매개변수 값은 undefined (Not 오류)
> * 매개변수보다 인수가 많은 경우 초과된 인수는 무시 (단, 버려지는 것은 아니고 암묵적 arguments 객체의 프로퍼티로 보관됨)

### 자바스크립트에서 함수 정의 시 적절한 인수가 전달되었는지 확인 필요 이유
1. 매개변수와 인수의 개수 일치 확인 X
2. 동적 타입 언어 -> 매개변수의 타입 사전 정의 X

* 매개변수는 순서에 의미가 있음
* 이상적인 매개변수 개수 = 0개, 적을수록 좋음
    * 매개변수는 최대 3개 권장!
* 이상적인 함수 = 한 가지 일만, 가급적 작게

### 반환문
* return + 표현식 => 함수 외부로 반환
* 반환문 역할
    1. 함수 실행 중단, 함수 몸체 탈출 -> 반환문 다음에 다른 문은 무시 (실행 X)
    2. return 키워드 뒤의 표현식 평가해서 반환. 명시적으로 평시 X 경우 `undefined` 반환
* 반환문 특징
    * 반환문 생략 시 암묵적 undefined 반환
    * return과 반환값 표현식 사이에 줄바꿈 있으면 의도치 않은 결과 발생 시킴
    * 함수 몸체 내부에서만 사용 가능

## 12.7
### 즉시 실행 함수
- 함수 정의와 동시에 즉시 호출, 단 한 번만 호출, 다시 호출 X
- 익명 함수 사용이 일반적, 그룹 연산자 내의 기명 함수는 함수 리터럴로 평가
- 반드시 그룹 연산자로 감싸야 함
- 일반 함수처럼 값 반환 O, 인수 전달 O

### 재귀 함수
- 자기 자신을 호출(재귀 호출)
- 반복되는 처리를 반복문없이 구현 가능
- 함수 표현식으로 정의한 함수 내부에서는 식별자로도 재귀 호출 가능
- 무한 재귀 호출로 `탈출 조건` 반드시 필요
    - 탈출 조건 없을 시 스택 오버플로우 발생

### 중첩 함수
- 함수 내부에 정의된 함수(=내부 함수)
- 외부 함수를 돕는 헬퍼 함수
    * 외부 함수 : 중첩 함수를 포함하는 함수

### 콜백 함수
- 함수의 매개변수를 통해 다른 함수의 내부로 전달
- 고차 함수에 전달되어 헬퍼 함수 역할

### 고차 함수
- 매개변수를 통해 함수의 외부에서 콜백 함수로 전달받음
- 콜백 함수를 자신의 일부분으로 합성

### 순수 함수
- 부수 효과 X
- 동일한 인수 전달 시 동일한 값 반환
- 매개변수 통해 함수 내부에 전달된 인수에만 의존 -> 값 생성, 반환
- 최소 하나 이상의 인수

### 비순수 함수
- 부수 효과 O
- 외부 상태에 따라 반환값 달라지는 함수
